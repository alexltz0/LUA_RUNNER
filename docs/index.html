Core Features

Built for Performance

Every component is designed for low-latency game event processing — from the sandboxed Lua VM to the fixed-timestep scheduler.

Sandboxed Execution

Dangerous globals (os, io, debug, require) are stripped. Lua scripts run in a restricted environment — zero escape vectors.

Event-Driven Architecture

High-performance pub/sub with priority queues, wildcard patterns (entity.*), one-shot listeners, and propagation control.

Fixed-Timestep Scheduler

Configurable tick rate (default 60 TPS) with frame budget tracking, scheduled tasks, and precise delta-time.

Entity System

Spawn, destroy, and modify game entities directly from Lua. Full property access with type-safe Python bridge.

Hot Reload

Reload all scripts without restarting the engine. Iterate on gameplay logic in real-time during development.

Performance Metrics

Built-in profiling: tick time, frame budget usage, event throughput, entity count — all accessible in real-time.

System Design

Layered Architecture

Lua scripts react to engine-driven events through a curated API surface. The inversion of control ensures stability and security.

Inversion of Control

Lua reacts to events — it never controls execution flow directly.

API Boundary

Engine exposes a curated set of functions. Scripts cannot corrupt engine state.

Lifecycle Hooks

on_init → on_tick → on_shutdown. Engine controls execution order.
